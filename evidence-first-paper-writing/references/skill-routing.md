# 科研 Skill 调用路由

本表是 `evidence-first-paper-writing` 的工作流路由，不替代任何下游 Skill 的详细说明。一次任务只调用当前阶段真正需要的 Skill。

各下游 Skill 的上游 GitHub 项目与维护者见 [上游开源项目署名](upstream-attribution.md)。

| 阶段或触发情况 | 调用的已有 Skill | 何时调用 | 预期交付物 |
|---|---|---|---|
| 只有方向、题目或模糊想法 | `academic-research-suite` | 尚无可回答的研究问题 | 候选研究问题、研究缺口、需验证假设 |
| 从零建立写作项目 | `researchwrite` | 需要把事实、证据、论证和章节先固定下来 | 研究基础、证据表、论证图、章节契约 |
| 需要检索、去重、引文核验或文献库 | `nature-academic-search` | 尚未形成可靠文献池，或要核对某一文献 | 检索式、已核验文献、RIS/BibTeX 等参考文献文件 |
| 要持续或批量发现新文献 | `nature-literature-pipeline` | 主题宽、来源多、需要评分筛选或长期跟踪 | 多源候选文献、评分、精读优先级和归档 |
| 需要下载可合法访问的全文 | `nature-downloader` | 作者已有学校/机构访问权限，或要处理访问失败 | 经授权获取的全文与访问状态说明 |
| 要精读或做中英对照文献 | `nature-reader` | 输入是 PDF、DOI、arXiv、出版社页面或全文 | 保留图表位置和来源锚点的中英对照精读稿 |
| 要把文献接入正文主张 | `nature-citation` | 某段正文需要严格、可追溯的引文支持 | 主张—引文对应、已核验参考文献输出 |
| 要写摘要、引言、方法、结果、讨论或全文 | `nature-writing` | 已有经过确认的主张、结果、图表、笔记或中文草稿 | 章节草稿、论文结构或完整论证文本 |
| 要把中文科研稿变为投稿级英文，或润色英文 | `nature-polishing` | 科学内容已确认，需要英文表达、重构或 LaTeX 版式处理 | 英文学术稿、修改说明或版式修复结果 |
| 要做中译英、英译中，并保留图表与来源位置 | `nature-reader` | 翻译对象为完整论文或文献 | 双语对应稿和来源锚点 |
| 要翻译或润色某一段科学正文 | `nature-polishing` | 翻译对象是论文段落、摘要、引言、方法或讨论 | 忠实的目标语段落与语言级修改 |
| 仅需减少成稿的 AI 模板味 | `humanizer` | 内容、术语、引文和结论均已确认 | 保持原意的自然表达版本与残余风险说明 |
| 要绘制、重做或审查论文图 | `nature-figure` | 已明确图要支持的结论、数据逻辑和目标格式 | 可投稿的 Python 或 R 科研图及质量检查结果 |
| 要准备数据、代码和共享声明 | `nature-data` | 数据集、代码、仓库、访问限制或投稿声明待处理 | 数据可用性声明、数据引用和 FAIR 检查表 |
| 要做投稿前模拟审稿 | `nature-reviewer` | 已有完整或接近完整的稿件 | 多视角审稿意见、问题优先级与综合判断 |
| 收到正式审稿意见，需要回复 | `nature-response` | 输入包含审稿意见、决定信或回复草稿 | 逐点回复信与修回行动项 |
| 收到正式审稿意见，需要核验并定点改稿 | `reviewer-driven-paper-revision` | 需要保留原稿、建立修订映射和证据核验 | 修订表、可替换文本、标记稿和回复依据 |
| 需要研究设计、统计解释、可复现性或全流程复核 | `academic-research-suite` | 方法风险、实验设计、完整性或研究阶段跨度较大 | 研究设计建议、完整性审计或端到端阶段计划 |
| 论文要做组会、答辩或会议汇报 | `nature-paper2ppt` | 已有论文、预印本、摘要或图表 | 中文学术 PPT、演讲备注和图表筛选 |
| 公开前考虑技术转化或专利 | `nature-paper-to-patent` | 需要识别可专利技术贡献，且尚未因公开失去策略空间 | 证据映射的中文专利草稿或专利风险判断 |

## 调用原则

1. 研究问题未定时，不调用正文写作或语言润色 Skill。
2. 没有作者确认的结果时，不调用任何会生成结果性叙述的流程。
3. 先用 `researchwrite` 固定证据和论证，再用 `nature-writing` 或 `nature-polishing` 写文本。
4. `humanizer` 必须位于内容、事实和引文核验之后；它只修语言，不修科学结论。
5. 论文进入外部审稿后，切换到返修流程，避免“从零写作”与“审稿意见驱动返修”混用。
