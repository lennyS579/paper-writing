# 证据优先的科学论文写作

从已有材料建立一篇可辩护的论文初稿。所有结论必须以证据为边界，不得把研究设想、预期、示例数字或未验证的解释写成已报告的发现。

## 与已有科研 Skill 的协作方式

本 Skill 负责从零写作的总流程、证据边界和阶段交付。遇到专门任务时，调用对应的已安装 Skill，而不是在本 Skill 内重复其工作。开始任务前读取 [科研 Skill 调用路由](references/skill-routing.md)，按“触发情况 → 调用 Skill → 交付物”执行。

调用前同时查阅 [依赖与许可证登记册](references/dependency-and-license-register.md)。该文件记录可选下游 Skill 的原始来源、许可证状态和再分发边界；不要把下游 Skill 的产出误署为本 Skill 独立开发。

- `academic-research-suite`：作为端到端研究、研究问题、实验设计、完整性核验和审稿流程的总控补充。
- `researchwrite`：作为证据表、论证图、章节契约和四层质量检查的写作底座。
- `nature-*`：作为文献、精读、引文、英文写作、图表、数据披露、预审、返修和成果转化的专业节点。
- `humanizer`：仅在科学内容确认后，用于去除模板化和空泛表达，不得改动科学事实。

若某个专用 Skill 不可用，保留同样的证据标准并说明降级处理；不得因为工具不可用而补造文献、数据或结果。

## 调度状态机

先判断当前状态，再执行对应阶段。不要因为用户直接要求“写论文”就跳过材料检查。

| 状态 | 判断 | 允许动作 | 禁止动作 |
|---|---|---|---|
| `S0_SCOPE` | 只有方向、题目或模糊想法 | 收敛研究问题和范围 | 写结果、完整摘要或确定性结论 |
| `S1_EVIDENCE` | 研究问题明确，但文献、数据或来源不足 | 建证据表、文献表和缺口清单 | 把未核验材料写成事实 |
| `S2_DESIGN` | 证据基础初步建立 | 固定研究对象、数据划分、基线、指标和统计方案 | 预设实验结果 |
| `S3_DRAFT` | 研究设计和可用证据已锁定 | 建论证图、章节契约并起草 | 隐去证据限制 |
| `S4_LANGUAGE` | 科学事实、术语、数据和引文已确认 | 翻译、润色和去模板化 | 改变结论强度或因果关系 |
| `S5_QA` | 稿件接近投稿 | 做引文、数字、图表、数据声明和预审检查 | 把待确认项当作已完成 |
| `S6_REVISION` | 收到审稿意见或决定信 | 切换逐点返修流程 | 把返修意见混入无关扩写 |

## 核心规则

- 先确定研究问题，再写提纲。只有宽泛方向时，先收敛问题，不直接起草摘要或正文。
- 明确区分已核验事实、作者假设、文献支持的主张、计划开展的工作和缺失证据。
- 先建立论证，再写段落。每个核心主张都要说明证据来源、适用限制和正文位置。
- 不得虚构结果、实验、效应量、引文、数据可用性、伦理审批、代码可用性或创新性结论。
- 方法和结果只能来自作者提供或确认的研究方案、数据、日志、图表、表格和记录。
- 证据不完整时保留不确定性，使用与证据强度相匹配的表述。
- 统一术语、单位、样本量、数据集划分和指标定义。
- 科学准确性优先于语言润色。只有内容、逻辑和证据通过检查后，才可进行中英翻译或减少 AI 味的语言处理。

## 工作流程

### 1. 明确研究框架

提取或询问：研究领域、现实或理论问题、候选贡献、目标期刊/会议、现有数据或实验、约束和截止时间。

没有可回答的研究问题时，给出 2 至 3 个候选问题，并分别说明需要的证据。未选定问题前，不产出完整提纲或摘要。

此阶段优先调用 `academic-research-suite` 的研究问题收敛或深度研究流程；已有研究方案、开题材料或模糊想法时，调用 `researchwrite` 的 compose 模式建立项目范围。

### 2. 建立研究基础包

建立以下工作表：

`研究问题 | 假设或目标 | 拟解决贡献 | 已有证据 | 缺失证据 | 范围边界 | 状态`

创建基础包前，读取 [研究基础模板](references/research-foundation-template.md)。缺少证据的结论标记为 `AUTHOR_INPUT_NEEDED` 或“待验证”，不得写成研究结果。

### 3. 建立文献与主张映射

记录每篇相关文献支持什么主张、采用何种方法或研究场景、有哪些局限，以及可在本文哪一处使用。文献综述必须围绕本研究确实能够回应的研究缺口组织。

不得引用未提供或未核验的文献；不得根据少量、未经核验的文献断言“首创”“最优”或“最新进展”。

需要检索、去重、核验 DOI/引文格式或管理 RIS/BibTeX 时调用 `nature-academic-search`；需要持续、多源、评分式文献发现时调用 `nature-literature-pipeline`；需要在作者已有权限下获取全文时调用 `nature-downloader`；需要保留图表位置的中英对照精读稿时调用 `nature-reader`。

### 4. 先写研究设计，再写结果

明确研究对象、数据来源、纳入/排除规则或数据集划分、方法、基线或对照、评价指标、统计方案和可复现材料。

结果尚未产生时，只能撰写研究方案、方法计划、表格框架和结果报告要求；不得预设结果走向。

### 5. 建立论证图与提纲

围绕论文中心论点建立章节地图：

`章节 | 写作目的 | 允许主张 | 所需证据 | 禁止主张 | 对应图/表 | 完成状态`

适用时采用“引言 → 相关工作 → 方法 → 结果 → 讨论 → 结论”的结构，但应优先服从目标期刊/会议规范。

使用 `researchwrite` 建立“主张—证据—章节契约”；需要从作者提供的结果、图表、笔记或中文草稿起草或重构具体章节时调用 `nature-writing`。

### 6. 按证据向外起草

优先顺序通常为：方法、结果、讨论、引言、摘要、标题。每段明确写出论点、证据、解释和限制。

仅可用 `AUTHOR_INPUT_NEEDED` 标记缺少的作者材料；提交版本中不得残留该标记或任何占位内容。

需要投稿级英文重写、英文润色、学术短语支持或 LaTeX 版式调整时调用 `nature-polishing`。需要为已写段落补充严格来源对应关系或管理参考文献输出时调用 `nature-citation`；引文必须先核验再写入正文。

### 7. 中译英、英译中与科学表达

翻译或英文润色前，读取 [学术中英互译与科学表达规范](references/academic-translation-and-style.md)。

- **中译英**：忠实还原科学含义、逻辑关系、限定条件、数字、单位、符号和引文锚点；使用学科中常见、可核验的英文术语，不为“高级感”扩写主张。
- **英译中**：忠实还原论证力度和不确定性，不把 `may`、`suggest`、`associated with` 等弱结论译为确定因果；保留公式、表图编号、单位和参考文献对应关系。
- **减少 AI 味**：仅在科学内容已确认后进行。删除空泛的意义拔高、模板化衔接、营销式形容词、无来源的“研究表明”和重复总结；保留必要的技术术语、限定语和被动语态。不得为了自然化而改变证据强度或方法事实。

输出译文时，同时提供：译文、术语对照、存在歧义或需作者确认的表达。除非用户要求，否则不擅自改变段落结构、引用格式或图表编号。

需要整篇论文、PDF 或带图表文献的中英对照阅读与翻译时优先调用 `nature-reader`；需要投稿级英文学术表达、中文稿转英文或 LaTeX 排版时调用 `nature-polishing`；仅对确认无误的成稿做减少 AI 味处理时调用 `humanizer`。

### 8. 进行论文完整性核验

读取 [起草与完整性检查清单](references/drafting-and-integrity-checklist.md)，逐项确认：

- 每项结果可追溯到源记录；
- 核心主张有充分且适度的支持；
- 方法足以让读者理解研究设计；
- 摘要、正文、图、表和图注中的数字一致；
- 引文直接支持其后的具体表述；
- 未回避研究局限和效度威胁；
- 伦理、数据、代码、利益冲突和 AI 使用声明准确，或明确待补。

需要数据可用性、代码共享、数据集引用、数据仓库或 FAIR 元数据时调用 `nature-data`；需要投稿前模拟审稿时调用 `nature-reviewer`；需要更全面的研究完整性、统计或实验复现性检查时调用 `academic-research-suite`。

### 9. 输出交付包

根据项目阶段交付：

- **研究计划**：研究问题、证据缺口、研究设计和里程碑。
- **写作基础包**：研究基础、文献映射、论证图、提纲和章节契约。
- **论文初稿**：带来源锚点的可直接使用文本，以及独立列出的作者待补项。
- **翻译与语言包**：目标语文本、术语表、歧义说明和科学表达审查结果。
- **投稿准备报告**：未关闭风险、需确认事项和最终检查清单。

收到真实审稿意见后，当前“从零写作”流程结束并切换至 `nature-response`；只有在用户单独提供了来源与许可证已核验的返修 Skill 时，才可进一步路由到该 Skill。将论文转为组会、答辩或会议汇报时调用 `nature-paper2ppt`；在公开论文或演讲前需要评估专利机会时调用 `nature-paper-to-patent`。

## 输出纪律

- 对关键内容标出确切来源、章节、表格、图或作者确认记录。
- 明确标识“已核验”“作者提供但未核验”“解释”和 `AUTHOR_INPUT_NEEDED`。
- 保留原始材料和记录，另建工作稿。
- 对外发布的示例或仓库不得包含私人论文、未发表数据、审稿材料、个人姓名、单位、文件路径或其他可识别信息。
